## SNAP Filesystem (LFS / DFS / AFS)

- LFS (`/lfs/<hostname>/0/brando9`): fastest, local to server, used as $HOME
- DFS (`/dfs/scratch0/brando9`): shared across all servers, use for repos + shared scripts
- AFS (`/afs/cs.stanford.edu/u/brando9`): slow, backed up, good for configs
- AutoFS: `cd /dfs/scratch0` to trigger mount if missing

## Kerberos Token Issue (critical for long runs)

- Problem: long tmux/Claude sessions cause Kerberos token expiration → Claude freezes
- Fix: open a new tmux session, run `reauth` (or `kinit`) to refresh tokens
- Important to document for anyone running overnight jobs on SNAP
